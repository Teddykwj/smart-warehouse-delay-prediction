# 공부 정리

## OOF Target Encoding

### Target Encoding이란?

범주형 컬럼의 각 카테고리 값을 **해당 카테고리의 타겟 평균값**으로 치환하는 인코딩 기법.

```
예시)
layout_id = "A" → 평균 지연시간 5.0분
layout_id = "B" → 평균 지연시간 2.3분

te__layout_id 컬럼:
  행이 layout_id=A이면 → 5.0
  행이 layout_id=B이면 → 2.3
```

트리 기반 모델(XGBoost, LightGBM)에서 Label Encoding보다 효과적인 경우가 많다.  
카테고리의 **서수 관계가 없어도** 타겟과의 관계를 수치로 잘 표현하기 때문.

---

### 문제: 데이터 누수 (Leakage)

학습 데이터 전체로 TE를 계산하면, 각 행의 타겟값이 **자기 자신의 피처 계산에 포함**된다.

```
train 전체로 layout_id=A의 평균 계산
  → 그 행 자신의 타겟도 평균에 포함됨
  → 모델이 "피처를 보면 타겟을 이미 알 수 있는" 상태가 됨
  → 학습 성능은 높지만 실전에서 급락 (과적합)
```

---

### 해결책: OOF (Out-Of-Fold) TE

Cross-Validation의 fold 구조를 활용해 **자기 자신의 타겟을 보지 않고** TE를 계산한다.

```
전체 데이터를 K개 fold로 분할

for each fold i:
    나머지 (K-1)개 fold → 인코더 학습 (train)
    fold i → 학습된 인코더로 변환 (자기 타겟 안 봄) (val)

test 데이터:
    train 전체로 인코더 학습 후 변환
    (test는 타겟이 없으므로 leakage 없음)
```

**시각화 (5-fold 예시)**

```
Fold 1 [────────] train
Fold 2 [────────] train  →  인코더 학습
Fold 3 [────────] train
Fold 4 [────────] train
Fold 5 [  val   ]        →  학습된 인코더로 변환 (leakage 없음)
```

각 fold가 돌아가며 val 역할을 하므로, 최종적으로 train 전체에 leakage 없는 TE가 채워진다.
-> val을 제외한 데이터의 타겟의 평균을 TE에 입력
---

### Unseen 카테고리 처리

test에 학습 데이터에 없던 카테고리가 등장하면 TE 값을 알 수 없다.  
→ **전체 타겟 평균(global mean)** 으로 대체 (`fillna(gmean)`)

정보가 없으면 중립값(평균)으로 처리한다는 합리적인 fallback.

---

### 콤보 TE

단일 키 외에 두 카테고리를 조합해 더 세밀한 패턴을 캡처할 수 있다.

```
layout_id + scenario_id → "A__scenario_1" 조합의 평균 지연시간
layout_cluster + layout_type → "cluster_3__type_B" 조합의 평균 지연시간
```

단, 조합 수가 많아질수록 각 카테고리 조합의 샘플 수가 줄어드는 **sparsity 문제** 주의.

---

### 핵심 요약

| 항목 | 내용 |
|---|---|
| 목적 | 범주형 변수를 타겟 기반 수치로 변환 |
| 문제 | naive TE는 leakage 발생 |
| 해결 | OOF — fold별로 자기 타겟 제외하고 계산 |
| Unseen 처리 | global mean으로 fallback |
| 응용 | 콤보 TE로 더 세밀한 패턴 캡처 가능 |

## XGBoost 하이퍼파라미터

### 1. 트리 구조 관련

**`max_depth`**

트리 하나가 얼마나 깊게 내려갈 수 있는지 정하는 값.

- 작다 → 단순한 규칙만 학습
- 크다 → 복잡한 조건 조합까지 학습

```
깊이 2 예시:
pressure < 20?
 ├─ yes: source_power < 500?
 └─ no: bias_power < 100?
```

깊이 10이면 조건을 훨씬 많이 이어 붙일 수 있어서 복잡한 패턴을 잡지만, 특정 데이터에만 맞는 규칙까지 외울 수 있음.

---

**`min_child_weight`**

분기 후 만들어지는 자식 노드가 가져야 하는 최소 데이터 무게.

> "이 정도 데이터도 안 모이면 굳이 가지를 만들지 마."

- 작음 → 세세하게 쪼갬 → 과적합 위험 증가
- 큼 → 충분한 데이터가 있을 때만 쪼갬 → 안정적

예를 들어 어떤 조건으로 나눴더니 한쪽에 데이터가 3개밖에 없다면, 그 3개를 위한 규칙은 위험할 수 있음. `min_child_weight`가 크면 그런 분기를 막아줌.

---

**`gamma`**

분기를 했을 때 손실이 얼마나 줄어야 그 분기를 인정할지 정하는 값.

- `gamma = 0` → 손실이 조금만 줄어도 분기 허용
- `gamma` 큼 → 손실이 많이 줄어야만 분기 허용

> "가지 하나 더 만들 거면, 성능 개선이 확실해야 해."

`gamma`가 커질수록 트리가 보수적으로 만들어짐.

---

### 2. 샘플링 관련

**`subsample`**

각 트리를 만들 때 전체 행 중 몇 %를 사용할지.

```
subsample = 0.8 → 매 트리마다 데이터의 80%만 랜덤하게 사용
```

- 낮음 → 랜덤성 증가, 과적합 감소, 하지만 정보 손실 가능
- 높음 → 안정적, 하지만 과적합 가능성 증가

---

**`colsample_bytree`**

각 트리를 만들 때 전체 피처 중 몇 %를 사용할지.

```
colsample_bytree = 0.6, 피처 100개
→ 트리 하나를 만들 때 60개 피처만 사용
```

모든 트리가 같은 강한 피처만 반복해서 쓰는 걸 줄여줌.

---

**`colsample_bylevel`**

트리의 각 깊이(level)마다 피처를 다시 샘플링. `colsample_bytree`보다 더 세밀한 랜덤성.

```
colsample_bytree = 0.8, colsample_bylevel = 0.8
→ 각 level에서 실제 사용 피처 비율 ≈ 0.8 × 0.8 = 0.64
```

---

### 3. 정규화 관련

**`reg_alpha`** (L1 정규화)

> "별로 중요하지 않은 리프 가중치는 아예 0에 가깝게 만들어."

- 작음 → 제약 약함
- 큼 → 일부 리프 영향 제거 (희소 모델)

피처가 많거나 노이즈가 많은 경우 유용.

---

**`reg_lambda`** (L2 정규화)

가중치를 0으로 만들기보다 **전체적으로 작게** 눌러줌.

> "각 리프의 예측값이 너무 커지지 않게 조심해."

| | reg_alpha | reg_lambda |
|---|---|---|
| 방식 | 필요 없는 건 끊어내기 | 전체적으로 힘을 줄이기 |
| 결과 | 희소 모델 | 전체 가중치 억제 |

너무 작으면 리프 값이 과하게 커질 수 있고, 너무 크면 모델이 둔해질 수 있음.

---

### 4. 학습 효율 관련

**`n_estimators`**

부스팅 트리를 최대 몇 개까지 만들지.

- 작음 → 충분히 못 배울 수 있음
- 큼 → 오래 걸리지만 early stopping 있으면 안전

`early_stopping_rounds`와 함께 쓰면 성능이 더 이상 좋아지지 않을 때 자동으로 멈추므로, 보통 넉넉히 크게 잡고 early stopping으로 최적 개수를 찾는 방식을 사용.

---

**`max_bin`**

`tree_method="hist"`에서 중요한 값. 연속형 피처를 몇 개의 구간으로 나눠서 볼지 결정.

```
어떤 피처가 0~100 범위일 때:
max_bin = 10  → 10개 구간으로 대충 나눔
max_bin = 512 → 512개 구간으로 정밀하게 나눔
```

- 작음 → 빠름, 거칠게 학습
- 큼 → 느림, 정밀하게 학습

---

### 한눈에 요약

| 파라미터 | 역할 | 크면 |
|---|---|---|
| `max_depth` | 트리 최대 깊이 | 복잡한 패턴, 과적합 위험 |
| `min_child_weight` | 분기 최소 샘플 무게 | 단순한 트리, 안정적 |
| `gamma` | 분기 허용 손실 임계값 | 보수적 분기 |
| `subsample` | 행 샘플링 비율 | 안정적, 과적합 가능 |
| `colsample_bytree` | 트리당 피처 비율 | 많은 피처 사용 |
| `colsample_bylevel` | 깊이당 피처 비율 | 많은 피처 사용 |
| `reg_alpha` | L1 정규화 | 가중치 희소화 |
| `reg_lambda` | L2 정규화 | 가중치 전반 억제 |
| `n_estimators` | 최대 트리 수 | 더 많이 학습 (early stopping 권장) |
| `max_bin` | 히스토그램 구간 수 | 정밀하지만 느림 |
